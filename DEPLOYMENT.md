# OMJI Deployment Guide

## Architecture Overview

```
Mobile App (React Native/Expo) → API (Go/Gin on Render) → PostgreSQL (Supabase)
Admin Dashboard (React/Vite) → Same API
```

## Prerequisites

- Node.js 20+
- Go 1.21+
- EAS CLI (`npm install -g eas-cli`)
- Expo account (expo.dev)
- Render account (render.com)
- Supabase project (supabase.com)
- Google Play Console (for Android)
- Apple Developer Account (for iOS)

---

## Backend Deployment (Render)

### First-time Setup
1. Create a new Web Service on Render
2. Connect your GitHub repo
3. Set build command: `go build -o bin/server cmd/main.go`
4. Set start command: `./bin/server`
5. Add a persistent disk (1GB) at `/var/data/uploads`

### Environment Variables (Required)
| Variable | Description | Example |
|----------|-------------|---------|
| `DATABASE_URL` | Supabase PostgreSQL connection string | `postgresql://...` |
| `JWT_SECRET` | Random 64-char string for JWT signing | Auto-generated by Render |
| `PORT` | Server port | `8080` |
| `GIN_MODE` | Gin framework mode | `release` |
| `ALLOWED_ORIGINS` | CORS origins (comma-separated) | `https://omji-admin.onrender.com` |
| `UPLOAD_DIR` | Upload directory path | `/var/data/uploads` |

### Health Check
- Path: `/health`
- Returns: server status, DB connection, uptime, pool stats

### Monitoring
- `/health` — public health check
- `/api/v1/admin/metrics` — detailed server metrics (admin auth required)

## Mobile App Deployment (EAS)

### Setup
```bash
cd mobile
npm install
eas login
eas build:configure
```

### Environment Variables
Set in EAS secrets (`eas secret:create`):
| Variable | Description |
|----------|-------------|
| `EXPO_PUBLIC_SENTRY_DSN` | Sentry DSN for crash reporting |

### Build Commands
```bash
# Development build (with dev tools)
eas build --platform android --profile development

# Preview APK (for testing)
eas build --platform android --profile apk

# Production build
eas build --platform android --profile production
eas build --platform ios --profile production

# Build both platforms
eas build --platform all --profile production
```

### Submit to Stores
```bash
# Google Play (requires google-play-key.json)
eas submit --platform android --profile production

# App Store (requires Apple credentials)
eas submit --platform ios --profile production
```

### OTA Updates
```bash
# Push a JS-only update (no native changes)
eas update --branch production --message "Bug fix description"
```

## Database (Supabase)

### Setup
1. Create project on supabase.com
2. Copy the connection string from Settings → Database
3. Set as `DATABASE_URL` in Render
4. Backend auto-migrates tables on startup via GORM

### Backups
- Supabase provides daily automatic backups (Pro plan)
- For manual backup: `pg_dump $DATABASE_URL > backup.sql`

## Running Locally

### Backend
```bash
cd backend
cp .env.example .env
# Edit .env with your local database credentials
go run cmd/main.go
```

### Mobile
```bash
cd mobile
npm install
npx expo start
# Press 'a' for Android, 'i' for iOS
```

### Admin
```bash
cd admin
npm install
npm run dev
```

## CI/CD Pipeline

GitHub Actions runs automatically on push to `main` and PRs:
- **mobile-tests**: Jest (125 tests)
- **backend-tests**: Go tests (14 tests)
- **mobile-typecheck**: TypeScript compilation
- **backend-build**: Go build verification

## Post-Launch Checklist

- [ ] Sentry DSN configured in EAS secrets
- [ ] JWT_SECRET is a strong random string (not default)
- [ ] ALLOWED_ORIGINS restricted to production domains
- [ ] Database backups enabled on Supabase
- [ ] Monitor Sentry for crash reports (first 48 hours)
- [ ] Monitor Render logs for API errors
- [ ] Test OTA update flow with `eas update`
- [ ] Verify push notifications work on production build
- [ ] Test GCash/Maya payment flow on real devices
- [ ] Submit to Play Store internal testing track first
